the program is currently only accessible through the browser console.

here is a list of functions to be used directly. those with a priority of 1 are the ones I think are most important and essential.
function name description priority
writeTask a simplified way to create a task object 1
showTable displays tasks in an organized table in the browser console 1
saveData* saves user data to the browser's local storage 1
removeTasks removes the corresponding task for each task id provided 1
getTasks returns the corresponding task object for each task id provided 1
markTasksAsCompleted sets the isCompleted property to true or false for each task provided 1
getTasksWithStatus returns an array of task objects with the status property value of either -1(overdue), 0(pending), or 1(completed) 1
sortTasksByStatus returns a copy of a provided array of task objects, sorted based on their status property value 1
getTasksWithWords returns an array of task objects which contain the terms provided 1
getTasksWithTags returns an array of task objects which contain all of the terms provided 1
getTasksByDueDate returns an array of task objects which are due within the date range provided 1
sortTasksByDueDate returns a copy of a provided array of task objects, sorted based on the date a task is due 2
Task.prototype.require stores ids of tasks that should be completed prior to the selected task without duplicating pre-existing entries 2
sortTasksByTags returns a copy of a provided array of task objects, sorted based on the amount of tags that match the terms provided 3
sortTasksByStartDate returns a copy of a provided array of task objects, sorted based on date of task creation 3
getTasksByStartDate returns an array of task objects which were created within the date range provided 3
getTasksWithNoTags returns an array of task objects which contain an empty tags array 4

*data not saved will be lost after a page refresh or when leaving the page.
